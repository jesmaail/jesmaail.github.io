Object Relational Mapping

In looking to improve an existing system developed as a C# WebSite, I was pointed in the direction of Object relational mapping or ORM.

  • Converts data between incompatible type systems in OOP
  • In effect, creates a "virtual object database"

  • Converts scalar Database values to non-scalar OO objects
    • e.g. An address book represents a single person, with zero or more phone numbers.
    This can be modelled as a Person object, holding a list of PhoneNumber objects.
In short, C# code using the SQL command:
"SELECT ... FROM persons WHERE id=10";
Can now become:
Person P = new repository.GetPerson(10);

Long Version
string sql = "SELECT ... FROM persons WHERE id=10";
DbCommand cmd = new DbCommand(connection, sql);
Result res = cmd.Execute();
String name = res[0]["First Name"];
Can now become:
Person p = repository.GetPerson(10); string name = p.GetFirstName();

In the context of C# .Net, this can be done using the Entity Framework.
The Entity framework is a set of technologies in ADO.Net which generates a 1:1 mapping between Database schema and conceptual schema.
