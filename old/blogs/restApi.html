Restful APIs

REST, or Representational State Transfer is a resource based architecture which must follow some constraints.

The six constraints something using the REST architecure must follow are:

  1. Uniform interface
  2. Stateless
  3. Client-Server
  4. Cacheable
  5. Layered system
  6. Code on demand

The Representations part of the REST architecture are:
  • Transferred between client and server.
  • Typically JSON or XML.
  • Example:
    • Resource: Person (Bob).
    • Service: Contact info (GET).
    • Representation: Name, Address, phone number (JSON/XML).

1. Uniform Interface

  • Defines interface between client and server.
  • Means:
    • HTTP verbs (GET, PUT POST, DELETE).
    • URIs (Resource name).
    • HTTP response (status, body).

2. Stateless

  • Server contains no client state.
  • Each request contains enough context to process message.
  • Any sessions state is held on the client.

3. Client-Server

  • Assume a disconnected system.
  • Seperation of concerns
  • Uniform interface is line between the two.

4. Cacheable

  • Server responses (representations) are cacheable
    • Implicitly
    • Explicitly
    • Negotiated

5. Layered system

  • Client can't assume direct connection to server.
  • Software or hardware intermediaries between client and server.
  • Improves scalability.

6. Code on demand

  • Server can temporarily extend client.
  • Transfer logic to client.
  • Client executes logic.
  • Example: Java applets, JavaScript.
  • The only optional constraint!
